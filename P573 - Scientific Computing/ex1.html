Clock Resolution and Overhead

Write two functions in C or C++ (or two subroutines in Fortran90+) that will get the timings needed to determine a clock's resolution and overhead. This can based by using the examples of how to call a clock, and they have makefile examples as well. Those two functions/subroutines must have the arguments

  • overhead(int* ndatapoints)
  • resolution(int* ndatapoints)
Yes, those are both pointers even though the data object is a single int. For Fortran the arguments are
  • overhead(ndatapoints)
  • resolution(ndatapoints)

    integer :: ndatapoints

Each of those will do the corresponding timer information algorithms shown in class and in the notes, as found on the overhead and resolution web pages. Each of the two functions will open a file, write the timing data to it, then close the files. The output from the overhead function must be named data_overhead, and for resolution the name must be data_resolution. Both files must be in plain ASCII, not binary.

In addition write a main function that

  • Reads in the number of data points (ndatapoints above) from a file named 'sizes'
  • Invokes the two functions overhead and resolution
  • Finishes
The main function must not do any I/O other than reading the sizes file. It can have print statements for debugging, but they must be turned off or removed when the assignment is turned in.

Provided files

  • show_overheads.m, a Matlab script that reads in the data file for overheads, and plots the data.
  • show_resolution.m, a Matlab script that reads in the data file for resolution finding, and plots the data.
  • analyze.m, used by show_resolution.m
  • enlargeaxis.m, used by show_resolution.m

Handin


Create a tar file named username-ex1.tar, where username is your email login name at IU. It must contain

  2. How many repetitions/sample data points are needed for finding clock facts?

    Short answer: as many as it takes. The recommended number of samples (aka number of data points or ndatapoints) is 200000, but while debugging and testing ndatapoints = 1000 is sufficient. The number of operations = noperations is in the notes about clock resolution . Once you know the resolution, the vector length n in finding the overhead should be relatively small. In class, it was suggested n = 2000 but explicitly stated to be something determined by experiment. If the clock resolution is high (i.e., 1 microsecond or less), n = 78 or 19 may suffice.
