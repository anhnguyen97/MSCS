Assignment 3 (100 points)

Announced 2012-04-09 (Mon) Due 2012-04-27 11:59 PM (Fri)

In this assignment you will implement a subset of OpenMP that we will call PidginMP. Your goal in this assignment is to implement PidginMP using a standard thread library and then compare the performance of the code your compiler emits to that compiled with a standard OpenMP compilers, such as gcc.

You have three tasks:

  1. Add support to parse PidginMP directives.
  2. Transform the PidginMP+PidginC program into standard C (without the PidginMP directives).
  3. Compile the code generated by your compiler with gcc and compare its performance with the original code compiled directly with gcc, this time with OpenMP support enabled within gcc (use -fopenmp option). Do this comparison for at least two distinct non-trivial examples. The algorithm for one example, Cholesky Factorization, is described in an accompanying document.

You will be evaluated based on the following:

  1. The correctness of your code, determined by running it on some test cases that will not be provided to you. Building an appropriate set of test cases is a part of the assignment.
  2. The collection of your own test cases that you submit with your solution. Add your tests to the test directory.
  3. The clarity of your code.

When you are ready to turn-in the assignment, use git tag -a to create a tag A3 and do not forget to push it back to the server. Then, send the instructor an e-mail and say if you are using additional time to finish your assignment. The instructor will extract the submission directly from your Git repository. Do not send a tar-ball of the entire code in e-mail.
